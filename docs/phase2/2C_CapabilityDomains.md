# 2C Capability Domains

## Document control

Objective: Record the factual capability domains visible from Phase 1 evidence.

Inputs: Phase 1B publications, Phase 1C projects, Phase 1D infrastructure,
Phase 1E collaborations, Phase 1F preliminary results, Phase 1G idea seeds,
Phase 1H bibliography records, and `docs/phase2/2_TeamBook.md`.

Outputs: One capability-domain map linked to stable Phase 1 evidence.

Dependencies: Phase 1B-1H stable IDs.

Completion criteria: Domains are linked to evidence records; no domain is
converted into an objective, gap, priority, responsibility, work package, or
proposal section.

Version: 1.0

Status: Active; initial subsection opened

Last update: 2026-07-01

This subsection records source-backed capability signals only. It does not
select project priorities or assign responsibilities.
