# 6 Master Project

## Document control

Objective: Build the master project from the completed Project Matrix and the
completed internal approval and budget-collection checkpoint.

Inputs: Project Matrix, Internal Approval and Budget Collection checkpoint, Gap
Analysis, Scientific Atlas, Team Book, and Phase 1 knowledge-base records.

Outputs: Reusable master project that can support multiple future funding
calls and the consolidated project package.

Dependencies: Completion of Phase 5.5.

Completion criteria: The master project is fully derived from prior controlled
documents and incorporates the approved budget, schedule, team information, and
project-owner accepted adjustments from Phase 5.5.

Version: 1.1

Status: Locked until Phase 5.5 is complete

Last update: 2026-07-02

## Scope guard

Do not write the CNPq proposal here. This document must remain reusable beyond
one funding agency.

Do not start this phase before the summarized proposal has been circulated to
project members, approvals/comments have been recorded, and budget items,
values, and justifications have been collected in Phase 5.5.
