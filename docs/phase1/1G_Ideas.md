# 1G Ideas

## Document control

Objective: Maintain the authoritative registry of research idea seeds
without turning them into objectives before Gap Analysis.

Inputs: Draft ideas, meeting notes, emails, conceptual sketches,
participant suggestions, preliminary-result records, publication records,
previous-project records, collaboration records, infrastructure records,
and source inventory entries.

Outputs: One reusable idea-seed record per idea.

Dependencies: `docs/phase1/1A_People.md`,
`docs/phase1/1B_Publications.md`,
`docs/phase1/1C_PreviousProjects.md`,
`docs/phase1/1D_Infrastructure.md`,
`docs/phase1/1E_Collaborations.md`, and
`docs/phase1/1F_PreliminaryResults.md`.

Completion criteria: Ideas are recorded once, with stable IDs, provenance,
and links to related people or evidence.

Version: 1.1

Status: Active; initial idea-seed registry opened

Last update: 2026-07-01

## Scope guard

This document is only an idea-seed registry.

Idea records are not scientific objectives. They are not selected project
directions. They do not imply novelty, priority, feasibility, importance,
funding strategy, team assignments, work packages, or CNPq proposal text.

Do not infer scientific objectives in this document. Objectives can only be
formed after Phase 4 - Gap Analysis. Work packages can only be created after
Phase 5 - Project Matrix. CNPq proposal writing can only start after Phase 6
- Master Project.

If an idea seems promising, keep it as an unselected seed and link it to the
evidence that made it visible. Do not promote it here.

## Phase gate note

Phase 1G was opened on 2026-07-01 after the Phase 1F preliminary-results
registry was reviewed for thematic coverage against Phase 1B publications.

Phase 1A through Phase 1F remain available for deferred manual refinement,
but they no longer block this initial idea-seed pass.

External collaborators, LAFMAT partners, ORNL facilities, and other
non-Phase-1A contexts may appear as evidence context only. Their appearance
here does not add them to the CNPq project team.

## Stable ID policy

Use stable idea IDs in the format `IDEA-0001`, `IDEA-0002`, `IDEA-0003`,
and so on.

Never renumber existing IDs. If a duplicate or obsolete record is found,
mark its status in the notes instead of reusing the ID.

## Required fields

Each idea-seed record must include:

- Unique ID
- Idea label

- Idea type
- Non-objective idea seed
