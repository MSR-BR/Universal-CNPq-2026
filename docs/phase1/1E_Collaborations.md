# 1E Collaborations

## Document control

Objective: Maintain the authoritative registry of collaborations, clearly
separating Phase 1A participant institutions from external collaboration
contexts.

Inputs: Collaboration agreements, coauthorship networks, institutional
partnerships, international links, meeting notes, CV collaboration statements,
previous-project records, and proposal collaboration sections.

Outputs: One reusable collaboration record per collaboration.

Dependencies: `docs/phase1/1A_People.md`,
`docs/phase1/1B_Publications.md`,
`docs/phase1/1C_PreviousProjects.md`, and
`docs/phase1/1D_Infrastructure.md`.

Completion criteria: Collaborations are recorded once, with stable IDs and
links to related people, institutions, projects, infrastructure, and
publications when available.

Version: 1.2

Status: Active; international collaborators added

Last update: 2026-07-01

## Scope guard

This document is only a collaboration registry.

A collaboration record does not make an external institution part of the CNPq
project team. Active project participants remain authoritative only in
`docs/phase1/1A_People.md`.

External collaborators, external institutions, LAFMAT nodes, and large-scale
facilities must be labelled as external/non-team unless a future Phase 1A update
adds them as participants. Antonio dos Santos and Paula Brandão were added to
Phase 1A on 2026-07-01 and are no longer treated as external people in this
registry.

Do not analyse scientific competencies here. Do not infer collaboration gaps.
Do not assign responsibilities. Do not create work packages. Do not write
proposal text.

## Phase gate note

Phase 1E was opened on 2026-07-01 after the Phase 1D infrastructure registry
was corrected to include only institutions represented in Phase 1A.

The project owner clarified that institutions appearing in the LAFMAT proposal
source do not automatically enter the CNPq project. This document therefore
records LAFMAT and other external institutions only as collaboration context,
not as team membership. This remains true for LAFMAT and remaining external
collaborators; it no longer applies to Antonio dos Santos and Paula Brandão
because they were explicitly added to Phase 1A.

## Stable ID policy

Use stable collaboration IDs in the format `COL-0001`, `COL-0002`,
`COL-0003`, and so on.

Never renumber existing IDs. If a duplicate, obsolete, or out-of-scope record
is found, mark its status in the notes instead of reusing the ID.

## Required fields

Each collaboration record must include:

- Unique ID
- Collaboration name
- Collaboration category
- Project participation status
- Phase 1A people involved
- Phase 1A institutions involved
- External people or institutions, if any
- Evidence stated in source
- Related projects
- Related infrastructure
- Related publications or outputs
- Source IDs
- Notes
