# 1F Preliminary Results

## Document control

Objective: Maintain the authoritative registry of preliminary results.

Inputs: Results summaries, datasets, figures, preprints, reports,
presentations, publication records, previous-project records, collaboration
records, infrastructure records, and source inventory entries.

Outputs: One reusable preliminary-result record per source-backed result or
result cluster.

Dependencies: `docs/phase1/1A_People.md`,
`docs/phase1/1B_Publications.md`, `docs/phase1/1C_PreviousProjects.md`,
`docs/phase1/1D_Infrastructure.md`, and
`docs/phase1/1E_Collaborations.md`.

Completion criteria: Preliminary results are recorded once, with stable IDs
and links to supporting evidence.

Version: 1.2

Status: Active; initial factual registry reviewed

Last update: 2026-07-01

## Scope guard

This document is only a preliminary-result registry.

Record only source-backed results, outputs, manuscripts, project-documented
work, and publication clusters that already exist in Phase 1 records or in
reviewed sources.

Do not convert preliminary results into objectives, hypotheses, scientific
gaps, proposal strategy, work packages, budget justification, or CNPq form
text. Do not rank importance or infer novelty beyond what the sources state.

Manual scientific review is deferred to the project-wide final review unless
the project owner requests it earlier.

## Phase gate note

Phase 1F was opened on 2026-07-01 after the initial Phase 1E collaboration
registry pass. Phase 1A through Phase 1E remain available for deferred manual
refinement, but they no longer block this factual preliminary-results pass.

External collaborators, LAFMAT partners, ORNL facilities, and other
non-Phase-1A contexts may appear as evidence context only. Their appearance
here does not add them to the CNPq project team.

## Stable ID policy

Use stable preliminary-result IDs in the format `RES-0001`, `RES-0002`,
`RES-0003`, and so on.

Never renumber existing IDs. If a duplicate or obsolete record is found, mark
its status in the notes instead of reusing the ID.

## Required fields

Each preliminary-result record must include:

- Unique ID
- Preliminary result label
- Evidence type
- Factual result statement
- Evidence basis
- Verification status
- Related publications
- Related projects
- Related people
- Related collaborations
- Related infrastructure
- Source IDs
- Notes
