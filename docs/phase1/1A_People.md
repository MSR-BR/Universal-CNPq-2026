# 1A People

## Document control

Objective: Build the structured database of every participant in the research
program.

Inputs: CVs, Lattes records, ORCID records, institutional pages, project
records, publications, and direct participant information.

Outputs: One stable participant record per researcher, grouped by institution.

Dependencies: `docs/01_Methodology.md` and `docs/00_Project_Status.md`.

Completion criteria: Every known participant has a stable ID and a complete
record with the required fields listed below.

Version: 1.0

Status: Active

Last update: 2026-07-01

## Scope guard

This document is only a People database.

Do not analyse scientific competencies here. Do not create work packages. Do
not assign responsibilities. Do not define scientific objectives.

## Stable ID policy

Use stable person IDs in the format `P-0001`, `P-0002`, `P-0003`, and so on.

Never renumber existing IDs. If a duplicate or obsolete record is found, mark
its status in the notes instead of reusing the ID.

## Required fields

Each researcher record must include:

- Unique ID
- Name
- Institution
- Department
- Position
- Academic degree
- Lattes
- ORCID, if available
- Current status
- Research group
- Career stage
- Related publications
- Related projects
- Notes
