# 3 Scientific Atlas

## Document control

Objective: Map the scientific landscape of the research program after the
Team Book baseline is accepted.

Inputs: Accepted Team Book baseline, Phase 2 subsection documents, Phase 1
knowledge-base records, and source inventory entries.

Outputs: Reusable scientific atlas with scientific areas, evidence anchors,
relationship maps, terminology notes, bibliography routes, and review items
for later controlled analysis.

Dependencies: Project-owner acceptance of Phase 2.

Completion criteria: Scientific areas, evidence, and relationships are mapped
from previous authoritative documents without creating scientific objectives,
gap analysis, responsibility assignments, work packages, budget items, or CNPq
proposal text.

Version: 1.2

Status: Active; detailed atlas pass completed; project-owner acceptance pending

Last update: 2026-07-02

## Scope guard

This document is the Scientific Atlas. It organizes the scientific landscape
visible in the accepted Team Book and Phase 1 evidence base.

Allowed work in Phase 3:

- organize scientific areas and evidence threads;
- map relationships among capability domains, evidence bundles, preliminary
  results, idea seeds, infrastructure, collaborations, and bibliography links;
- clarify terminology and boundaries among scientific areas;
- preserve uncertainty and manual-review notes for later checking;
- prepare neutral atlas material that can support Phase 4 - Gap Analysis.

Still prohibited:

- do not define scientific gaps;
- do not select proposal objectives;
- do not rank themes as priorities;
- do not assign people, institutions, or infrastructure to responsibilities;
- do not create work packages, schedule, budget, or access commitments;
- do not write CNPq proposal text.

## Atlas method

Phase 3 translates Phase 2 capability domains and Phase 2F evidence bundles
into neutral scientific atlas areas.

The atlas uses four classes of records:

| Record class | Local codes | Role in Phase 3 |
| --- | --- | --- |
| Scientific atlas areas | `SA-*` | Organize the scientific landscape visible in the evidence base. |
| Relationship routes | `RR-*` | Map factual scientific connections among atlas areas. |
| Terminology notes | `TERM-*` | Keep vocabulary stable before public-facing writing. |
| Review carry-over items | `REV-*` | Preserve manual checks without blocking the atlas. |

These local Phase 3 codes do not replace stable Phase 1 IDs.
