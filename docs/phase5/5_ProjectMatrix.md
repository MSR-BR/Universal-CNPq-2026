# 5 Project Matrix

## Document control

Objective: Convert validated gaps into a reusable project matrix.

Inputs: Gap Analysis, Scientific Atlas, Team Book, and Phase 1 knowledge-base
records.

Outputs: Project matrix for future internal approval, budget collection,
master-project, and proposal development.

Dependencies: Completion of Phase 4.

Completion criteria: Project options, relationships, and priorities are
structured from the completed Gap Analysis and are ready to support Phase 5.5
internal approval and budget collection.

Version: 1.1

Status: Locked until Phase 4 is complete

Last update: 2026-07-02

## Scope guard

Do not create work packages before this phase.

Do not start Phase 6 directly from this document. Phase 5.5 must first
circulate the summarized proposal to project members, record approval/comments,
and collect budget information.
